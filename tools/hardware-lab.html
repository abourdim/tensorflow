🎮 Gesture Classifier
micro:bit v2Beginner30 min

Train AI to recognize physical gestures — shake, tilt, spin, wave. Your body movements become training data that a neural network learns from!

How It Works

Connect micro:bit to your computer via Web Bluetooth (Chrome/Edge required)
Choose 3-4 gestures to train (e.g., shake, tilt left, tilt right, spin)
Record 10 samples of each gesture — the accelerometer captures x, y, z movement data
TensorFlow.js trains a small neural network on your gesture data right in the browser
Test: perform a gesture → the AI predicts which one you did
Experiment: What happens with fewer samples? Similar gestures? Faster movements?

Parts Needed

micro:bit v2
USB cable
Chrome or Edge browser

No wiring needed — uses built-in accelerometer

Code Example

// Connect to micro:bit via Web Bluetooth
async function connectMicrobit() {
  const device = await navigator.bluetooth
    .requestDevice({
      filters: [{ namePrefix: 'BBC micro:bit' }],
      optionalServices: ['e95d0753-251d-470a-a062-fa1922dfa9a8']
    });
  const server = await device.gatt.connect();
  // Read accelerometer data
  const service = await server
    .getPrimaryService('e95d0753...');
  // Stream x, y, z values to TensorFlow.js
}

"Your physical movements became numbers, the numbers became patterns, and the AI learned those patterns. This is how fitness trackers and game controllers work!"

🌡️ Sensor Data Sorting
micro:bit v2Beginner25 min

Collect real environmental data from different locations and classify readings. Experience how messy real-world data is compared to clean training sets.

How It Works

Connect micro:bit and stream temperature + light sensor data to the browser
Walk around and take readings: near a window, under a desk, near a heater, outside
Label each reading: bright/dark, hot/cold, indoor/outdoor
See how a simple classifier would sort new, unlabeled readings
Discussion: "What features did YOU use to classify? Temperature? Light? Both?"

Parts Needed

micro:bit v2
USB cable
Chrome or Edge browser

No wiring — uses built-in temperature and light sensors

"Real-world data is messy, noisy, and sometimes contradictory. That's what makes AI engineering challenging — and exciting. Clean data is a luxury!"

🌱 Smart Plant Monitor
ESP32-C3Intermediate45 min

Build a real IoT device that monitors plant health. Soil moisture, light, and temperature sensors feed data to an AI that classifies: healthy, needs water, needs light, or too hot.

How It Works

Wire soil moisture, light (LDR), and temperature sensors to ESP32-C3
ESP32-C3 hosts a web server with a live sensor dashboard
Collect data from healthy plants and stressed plants (or simulate conditions)
Train a classifier in the browser: healthy / needs water / needs light / too hot
Deploy: ESP32-C3 continuously monitors and shows plant status on its web page

Parts Needed

ESP32-C3 dev board
Soil moisture sensor
LDR light sensor
Temperature sensor (DHT11/22)
Breadboard
Jumper wires
10kΩ resistor (for LDR)

Wiring Diagram

ESP32-C3 wiring: 3 sensors → analog/digital pins → WiFi web dashboard

"You built a real IoT + AI product. Smart agriculture uses exactly this approach — sensors collecting data, AI making decisions, farmers saving crops."

⚡ Edge AI Deployment
ESP32-C3Advanced60 min

The ultimate challenge: train a model in the browser, convert it to TensorFlow Lite, and deploy it to run ON the ESP32-C3. No internet needed — the AI lives on the chip.

How It Works

Train a model in Workshop 3 (or build a new one in the browser)
Export the model to TensorFlow Lite format (quantized for microcontrollers)
Flash the TFLite model to ESP32-C3 using Web Serial API
Connect a sensor to the ESP32-C3 (e.g., accelerometer for gesture recognition)
The ESP32-C3 runs inference locally — no WiFi, no cloud, no internet needed
Measure inference speed: how many predictions per second can this tiny chip do?

Parts Needed

ESP32-C3 dev board
USB-C cable
Chrome or Edge browser
Optional: sensor for live inference

Key Concepts

Cloud AI
Runs on big servers. Needs internet. Powerful but adds latency.
Edge AI
Runs on tiny chips. No internet. Fast but limited power.

Why Edge AI matters: Privacy (data stays on device), speed (no network latency), reliability (works offline), cost (no cloud bills).

"Cloud AI needs internet. Edge AI doesn't. Both have tradeoffs — now you understand why. You just did what real IoT engineers do every day."
